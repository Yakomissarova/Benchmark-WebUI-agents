
# Web-UI-Agent

## Запуск:

1. Добавить в `.env`:

   * `OPENAI_API_KEY=ваш_ключ`  (обязательно)
   * `BROWSER_USE_API_KEY=ваш_ключ` *(опционально, для browser-use API модели)*
   * `OPEN_ROUTER_MODEL=model` *(опционально, используется как default если не передан --model)*

---

2. Установить зависимости:
   `pip install -r requirements.txt`

---

3. Распаковать большие HAR записи из ZIP файлов в папке `benchmark_data`

---

4. Установить браузеры:

   * `python -m playwright install chromium`
   * `uvx browser-use install`

---

5. Запустить бенчмарк из корня проекта:

### 🔹 OpenRouter агент

```bash
python -m benchmark.benchmark_runner --agent openrouter --model deepseek/deepseek-chat-v3.1
```

Модель задаётся через `--model`
Если не указать — возьмётся из `.env`

---

### 🔹 Browser-use агент

1. **Сначала запустить Chrome с CDP:**

```bash
"C:\Program Files\Google\Chrome\Application\chrome.exe" ^
  --remote-debugging-port=9222 ^
  --user-data-dir="C:\temp\chrome-debug"
```

2. Затем запустить бенчмарк:

```bash
python -m benchmark.benchmark_runner ^
  --agent browser_use ^
  --model deepseek/deepseek-chat-v3.1 ^
---
```
### Ограничить количество сценариев (для теста)

```bash
--limit 3
```

---

## Результаты

Результаты каждого прогона записываются в:

```text
benchmark_results.json
```

Каждый новый запуск **добавляется**, а не перезаписывает файл.

В результатах сохраняется:

* агент и модель
* метрики
* подробный трейс действий

---
## Структура репозитория

### `benchmark/`

Управляющий модуль. CLI принимает `--agent`, `--model`, `--limit`, читает `benchmark_data/benchmarks.json`, поднимает Chromium с подключённым HAR-хранилищем, прогоняет агента по задачам и считает метрики из подсистемы оценки.


### `benchmark_data/`

Данные бенчмарка:

- `benchmarks.json` — описания 60 задач. Поля: `id`, `name`, `har_path`, `start_url`, `type`, `difficulty`, `goal`, `eval_type`, `eval_value`, `checkpoints`, `trace`.
- HAR-копии девяти сайтов: РЖД, Эрмитаж, Отелло, Суточно, Яндекс Афиша, КонсультантПлюс, Sunlight, Citilink, Lamoda. Большие файлы лежат в ZIP, перед запуском их нужно распаковать.

### `manual_parsing/`

Скрипты для ручного сбора датасета:

- `playwright_recording.py` — запись HAR-копии сайта. Открывает Chromium с включённой записью HAR, ждёт ручного прохода сценария оператором, фиксирует финальный URL и сохраняет метаданные в `recordings.json`.
- `playwright_reading.py` — воспроизведение HAR без обращения к сети, для проверки записанных копий.
- `collect_url.py` — сбор последовательности контрольных URL для эталонной траектории задачи.

### `benchmark_analysis.ipynb`

Разбор `benchmark_results.json`: сводные таблицы по сайтам, разрезы по уровням сложности и типам задач, графики Success Rate.

## Состав датасета

60 задач на 9 русскоязычных сайтах. Каждая описана отдельным объектом в `benchmark_data/benchmarks.json`.

**Уровни сложности** (по числу действий в эталонной траектории):

- `easy` — 1-4 действия, навигация и поиск
- `medium` — 5-9 действий, формы и фильтры
- `hard` — 10+ действий, полный сценарий от поиска до оформления

**Типы задач**:

- `travel` — бронирование и билеты (РЖД, Отелло, Суточно)
- `entertainment` — культурные события (Эрмитаж, Яндекс Афиша)
- `info` — поиск информации (РЖД, Эрмитаж, Суточно, КонсультантПлюс)
- `shopping` — подбор товара (Sunlight, Citilink, Lamoda)

**Типы автопроверки результата**:

- `url_contains` — итоговый URL содержит подстроку
- `url_pattern` — URL подходит под регулярное выражение
- `element_exists` — на странице есть селектор
- `text_contains` — на странице есть фрагмент текста

---

## Метрики

Считаются раннером после каждого прогона и пишутся в `benchmark_results.json`:

- `Success Rate` — доля выполненных задач
- `Visited Rate` / `Stopped Correctly` — попал ли агент на целевой URL и завершил ли работу именно на нём
- `Avg Progress` — средний прогресс по контрольным URL из `checkpoints`
- Среднее число шагов (по всем задачам и отдельно по успешным)
- Средняя длительность выполнения
- Суммарное число токенов
- `ErrRatio` — доля шагов с ошибками
- `UselessRatio` — доля шагов, не изменивших состояние страницы

## Запись HAR файлов:

1. В `playwright_recording.py` вставить название файла для записи и ссылку на сайт, чтобы playwright его открыл.
2. Запустить скрипт, откроется браузер chromium.
3. Проделать действия, которые должен будет проделать агент.
4. Закрыть chromium, файл сохранится в папку benchmark_data.

---

## Чтение HAR файлов

1. В `playwright_reading.py` вставить название файла для записи и ссылку на сайт, чтобы playwright его открыл.
2. Запустить скрипт, откроется браузер.
3. Можно ходить по сайту в пределах записанных страниц.
